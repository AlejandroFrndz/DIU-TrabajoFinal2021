## Parte II: Caso de Estudio. YUZIN
### Paso 1. UX Desk Research & Analisis
Puesto que el objetivo del trabajo es realizar un rediseño de la web, lo primero que debemos hacer es analizar la solución actual, es decir, la web que nos presenta [YUZIN](https://yuzin.com/). Uno de los aspectos que nos presenta el trabajo es que la web ya experimentó recientemente un rediseño en el que se modificó tanto el aspecto visual de la web como su foco principal. Antes la web se centraba en la revista que Yuzin publica mensualmente en la que recogen eventos culturales en Granada y pasó a tener una orientación más digital, mostrando la web actual, y dejando la revista en un segundo plano. Por tanto, para conocer con qué trabajamos vamos a utilizar la herramienta [Wayback Machine](https://web.archive.org/) que nos permitirá ver de primera mano la versión antigua y así podremos comprender mejor la historia del sitio y lo que suponen los cambios realizados\
![Antigua Web YUZIN](imgs/yuzin_antigua.png)
Podemos ver que se trata de una web completamente estática, lo cuál está completamente desfasado con respecto a las tendencias actuales. Si bien no se debe sobrecargar de dinamismo una web, tampoco es bueno que esta sea completamente estática. Con el diseño antiguo, la web era una herramienta de paso en la que los usuarios apenas pasarían tiempo pues no se les ofrece nada más que un escaparate mediante el cual acceder a la revista. Por tanto, considero adecuado haber realizado un rediseño de la web para aportarle mayor funcionalidad y en definitiva aportar algo de valor que haga que los usuarios quieran acceder a la web y pasar algo de tiempo en ella. No obstante, habrá que estudiar ahora como de bueno es el rediseño efectuado, identificar qué aspectos se podrían mejorar y finalmente exponer mi propuesta de (re)rediseño.\
Para el análisis de la propuesta actual se realizarán 3 tareas. Actuando como experto en UX, realizaré una evaluación heurística de la web para identificar los puntos fuertes y débiles. También se creará una persona ficticia que mediante un Journey Map usará la web y expondrá su experiencia. Finalmente, también se realizará una evaluación de la accesibilidad de la solución actual, pues también es un factor fundamental a considerar.

### Paso 2. UX [Re]Design
#### 2.1 Logo y Paleta de Colores
Creo que el logo que se usa actualmente es adecuado, dando un cierto estilo que recuerda a una escritura manuscrita que puede evocar a la revista que tradicionalmente ha sido yuzin. Sin embargo, la paleta de colores presenta un claro problema de contraste. El uso del color blanco de fondo para la web me parece adecuado ya que sirve para no sobrecargar visualmente la página y deja espacio para que el diseño respire. Ahora bien, el color de acento creo que debería cambiarse, pues el uso del amarillo que actualmente proponen no da un contraste suficiente con el fondo. Por lo tanto, propongo un cambio a la paleta de colores siguiente ![Paleta de Colores](imgs/ColorPalette.png) (Creada mediante la herramienta [Coolors](https://coolors.co/))\
Se mantiene el uso del blanco para el fondo de la página y el negro para el grueso general del texto. Sin embargo, se sustituye el amarillo por el carmesí 610522 para los elementos destacados, de un tono más oscuro y que ofrecerá mejor contraste con el fondo. También se propone el color naranja E58938 como color acento alternativo con el que poder resaltar las opciones destacadas que sean seleccionadas, por ejemplo en la navegación por teclado, para dejar claro al usuario las opciones que está seleccionando. Coolors ofrece además una herramienta para simular distintos tipos de daltonismo gracias a la cual he podido comprobar que la paleta ofrece suficiente contraste para todo tipo de usuarios. #### 2.2 SiteMap
Aunque en general la página tiene una buena estructura si que hay un par de aspectos que podrían mejorarse. Por un lado se debería crear una sección para los artículos de opinión que incluye la página. De esta forma habría un lugar en el que poder recogerlos todos de forma que la gente que entre a la web buscandolos tenga un sitio donde poder encontrarlos fácilmente, en el que también se podría implementar un buscador para facilitar la búsqueda de un artículo concreto. De esa forma se podría también dejar 1 solo artículo destacado en la página principal y mover el resto a su sección correspondiente, liberando parte de la carga de la página principal. Aunque entiendo que no es el focus principal de la web ahora, si es algo que distingue esta web del resto que podría darle una identidad propia.
